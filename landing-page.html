Flask by Example

Published: Atl. Web Exch. | Jan. 17, 2020

Landing Page

Welcome! The objective of this tutorial is to build a Flask app that calculates word-frequency pairs based on the text from a given URL.

This Series's Procedural Outline

  • Step One: Set up a local development environment and then deploy both a staging and a production environment on Heroku. (current)
  • Step Two: Set up a PostgreSQL database along with SQLAlchemy and Alembic to handle migrations.
  • Step Three: Add in the back-end logic to scrape and then process the word counts from a webpage using the requests, BeautifulSoup, and Natural Language Toolkit (NLTK) libraries.
  • Step Four: Implement a Redis task queue to handle the text processing.
  • Step Five: Set up Angular on the front-end to continuously poll the back-end to see if the request is done processing.
  • Step Six: Push to the staging server on Heroku - setting up Redis and detailing how to run two processes (web and worker) on a single Dyno.
  • Step Seven: Update the front-end to make it more user-friendly.
  • Step Eight: Create a custom Angular Directive to display a frequency distribution chart using JavaScript and D3.

Tutorial Updates:
